# 中文创作者研究提示词

## 任务

对一个中文短视频创作者做结构化研究，为生成 Creator Skill 做准备。

目标不是写人物传记，也不是模仿口头禅，而是从公开或授权材料中提取：

- 选题系统：他/她通常选择什么议题，为什么这些议题适合这个账号
- 内容结构：短视频如何开头、推进、转折和收束
- 表达 DNA：语言节奏、比喻来源、判断方式、情绪强度
- 判断模型：面对一个新选题时会如何取舍
- 反模式：明确拒绝什么内容、什么表达、什么立场
- 安全边界：哪些地方证据不足，哪些地方不能生成

## 材料优先级

对于 Thousand Faces Style Skill，TikHub 下载和阿里云 ASR 已经在研究前完成。优先使用运行目录中的本地材料：

1. `research/host_refinement/brief.md`：宿主研究入口，包含标题索引、高互动样本、主题分布和代表性转写片段
2. `transcripts/*.txt`：核心材料，需要按视频 ID 有选择地读取
3. `metadata/selected.compact.json`：视频标题、发布时间、统计数据、源链接
4. `research/merged/summary.md`：确定性摘要，只作为索引，不替代原转写稿
5. 用户额外提供的本地材料：PDF、截图、笔记、人工修正转写
6. 外部公开来源：长访谈、公开文章、B 站/播客长内容、官方账号原文

不要把平台首页、搜索页、话题页、个人主页根地址当作证据。只有实际打开并阅读过的具体内容 URL 才算证据。

## 来源质量

优先级从高到低：

1. 用户提供或流水线生成的本地转写稿
2. 创作者本人发布的长内容、长访谈、直播回放、播客
3. 创作者本人发布的短视频、微博、图文
4. 有明确出处的媒体采访或报道
5. 他人分析、评论、剪辑
6. 二手总结和搬运内容

不要引用：

- 百度百科
- 未注明来源的公众号搬运
- 无法核验的知乎匿名回答
- AI 自动生成的人物页
- 只做 SEO 的“十大经验”类文章

## 六个研究维度

不要写成一篇泛泛的人物介绍。按维度做笔记，每个维度都要区分“证据”和“推断”。

### 1. 选题系统

关注：

- 常见选题领域
- 选题背后的冲突或问题意识
- 哪些题材反复出现
- 哪些题材虽然热门但该创作者不碰
- 选题是否依赖热点、产品演示、行业判断、个人经验或价值立场

输出：

- 3-7 个选题簇
- 每个选题簇的适用条件
- 每个选题簇的证据视频

### 2. 脚本结构

关注：

- 开头 3-10 秒如何抓注意力
- 主体如何分段
- 是否常用演示、对比、反转、故事、清单、案例
- 结尾如何收束：观点、行动号召、情绪落点、悬念
- 视频节奏：密集信息、慢解释、连续抛梗、强观点推进

输出：

- 常见脚本模板
- 每种模板的适用场景
- 证据视频和对应片段摘要

### 3. 表达 DNA

关注：

- 语气：克制、兴奋、吐槽、教学、朋友式、评测式、判断式
- 句式：短句密集、长句解释、反问、排比、转折
- 词汇：专业术语、口语词、行业黑话、固定表达
- 比喻来源：产品、工程、商业、生活、游戏、职场、科学等
- 情绪强度：冷静分析、强烈推荐、警惕提醒、幽默拆解
- 确信度表达：如何说“我确定”，如何说“我不确定”

输出：

- 可识别的表达标记
- 不能过度模仿的口头习惯
- 适合复用的表达原则

### 4. 判断模型

关注：

- 评价一件事时先看什么
- 判断好坏的标准是什么
- 什么证据会改变结论
- 什么情况下会保守，什么情况下会下注
- 常见权衡：效率 vs 质量、创新 vs 稳定、短期热度 vs 长期价值

输出：

- 3-5 个判断模型
- 每个模型的适用边界
- 每个模型的失败模式

### 5. 反模式与边界

关注：

- 创作者反复批评什么
- 哪些表达方式不符合该账号
- 哪些话题证据不足
- 哪些生成请求应该拒绝
- 哪些地方只能说“基于样本推断”

输出：

- 内容反模式
- 表达反模式
- 安全边界
- 证据薄弱点

### 6. 演化时间线

关注：

- 最近 12-24 个月选题是否变化
- 表达方式是否变化
- 内容定位是否变化
- 是否出现新的固定栏目、系列或观点

输出：

- 阶段划分
- 明显变化
- 仍然稳定的核心模式

## 原始研究笔记产物

至少生成 5 个原始研究笔记文件，放到 `research/raw/`：

- `01_topic_and_timeline.md`：选题系统 + 演化时间线
- `02_structure_and_judgment.md`：脚本结构 + 判断模型
- `03_expression_and_boundary.md`：表达 DNA + 反模式与边界
- `04_contradictions_and_evolution.md`：反例、矛盾、阶段变化和证据缺口
- `05_short_form_patterns.md`：短视频 hook、快速转折、结尾收束和短样本证据强度

每个文件使用以下结构：

```md
# <研究维度标题>

## 覆盖范围
- 覆盖维度：
- 使用材料：local transcripts / metadata / web sources

## 来源元数据
- 文件或 URL：
- 来源类型：转写稿 / 视频元数据 / 访谈 / 文章 / 用户修正
- 证据等级：一手 / 二手 / 推断
- 权重：1-7

## 关键发现
- ...

## 重复模式
- ...

## 矛盾与变化
- ...

## 推断
- 标明这是推断，不要写成事实

## 缺口
- ...
```

## 质量底线

整轮研究至少满足：

- 至少 5 个 raw note 文件
- 每个文件都区分证据和推断
- 至少覆盖 15 个视频锚点；样本数超过 50 时，优先覆盖高互动、长转写、典型结构、边界/反模式样本
- 至少识别 1 个反模式或安全边界
- 至少承认 1 个证据缺口
- 必须为后续 `persona_model.json` 提供 topic model、script template、judgment heuristic、expression DNA、anti-pattern、安全边界和 evidence anchor 的候选材料
- 必须使用 `coverage_gaps.md` 补读或解释高优先级缺口视频
- 必须为后续 `evaluation_suite.md/json` 和 `reverse_identification.md/json` 提供可测试的任务场景、反例和证据锚点
- 避免“引发共鸣”“层层递进”“通俗易懂”等泛化模板话术，改写为可追溯的视频证据和具体规则
- 不把完整转写稿复制进 skill 目录
- 不长篇引用字幕或转写稿

## 研究完成后的检查摘要

在进入分析和构建前，输出：

```text
维度 | 证据数量 | 关键发现 | 证据缺口
选题系统 | N | ... | ...
脚本结构 | N | ... | ...
表达 DNA | N | ... | ...
判断模型 | N | ... | ...
反模式与边界 | N | ... | ...
时间线变化 | N | ... | ...

是否样本过少：是/否
需要用户补充的材料：...
是否可以进入 Skill 构建：是/否
```

## 输出规则

- 使用中文。
- 明确区分：创作者说过什么、创作者做过什么、他人怎么看、你推断了什么。
- 矛盾不要抹平，保留为风格复杂性的一部分。
- 宁可承认证据不足，不要补泛泛而谈的内容。
- 不要编造来源、URL、标题、时间或引用。
- 不要把完整转写稿或大段字幕放进生成的 skill。
- 直接引用必须短，且只在捕捉表达 DNA 时使用。
